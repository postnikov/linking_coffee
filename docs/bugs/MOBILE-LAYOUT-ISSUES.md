## üêõ Bug 1: Glass-card overflow on mobile

**Severity:** High (visual breakage)

**Problem:**
`.glass-card` elements are wider than their parent `.dashboard-container`, causing horizontal overflow and a dark stripe visible on the right side of the screen.

**Measurements at 440px viewport:**
| Element | Width | Expected |
|---------|-------|----------|
| `viewport` | 440px | ‚Äî |
| `body` (with scrollbar) | 432px | ‚Äî |
| `.dashboard-main` | 432px | ‚úÖ |
| `.dashboard-container` | 384px | ‚úÖ |
| `.glass-card` | 419px + 8px margin = **427px** | ‚ùå Should be ‚â§368px |

**Overflow detected:**
- `.dashboard-main` scrollWidth: **459px** vs clientWidth: **432px**
- Content overflows by ~27px

**Root cause:**
In `Dashboard.css` @ line 244-247:
```css
@media (max-width: 640px) {
  .profile-section.glass-card,
  .dashboard-container .glass-card {
    margin: 0 0.25rem;
    width: calc(100% - 0.5rem);
  }
}
```

The `width: calc(100% - 0.5rem)` calculates relative to the wrong parent, or there's a CSS specificity conflict with `App.css` glass-card styles.

**Suggested fix:**
```css
@media (max-width: 640px) {
  .dashboard-container .glass-card {
    width: 100%;
    max-width: 100%;
    margin: 0;
    box-sizing: border-box;
  }
}
```

## üêõ Bug 2: Forced scrollbar steals viewport width

**Severity:** Low (cosmetic)

**Problem:**
`html { overflow-y: scroll }` in `App.css` forces a permanent vertical scrollbar, reducing effective viewport from 440px to 432px.

**Impact:**
- 8px "dark stripe" on right side (gradient background visible)
- Less usable width for content

**File:** `/frontend/src/App.css` line 1-3
```css
html {
    overflow-y: scroll;
}
```

**Suggested fix:**
Consider using `overflow-y: auto` or overlay scrollbars for mobile:
```css
@media (max-width: 768px) {
  html {
    overflow-y: auto;
  }
}
```

## üêõ Bug 3: Dashboard container padding inconsistency

**Severity:** Medium

**Problem:**
`.dashboard-main` has `padding: 76px 24px 48px` (24px horizontal), but `.dashboard-container` inside has additional `padding: 16px 8px`. Combined with card margins, this creates uneven spacing.

**Current state:**
- Main: 24px padding left/right
- Container: 8px padding left/right
- Cards: 4px margin left/right
- Total: 36px on each side ‚Äî excessive for 440px screen

**Suggested fix:**
Reduce padding on mobile:
```css
@media (max-width: 640px) {
  .dashboard-main {
    padding-left: 8px;
    padding-right: 8px;
  }
  
  .dashboard-container {
    padding: 1rem 0;
  }
}
```
